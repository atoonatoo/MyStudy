### CS Knowledge Base

#### 1. 네트워크 통신 기초

##### 데이터그램 (Datagram)

- 독립적인 봉투
    - 수신 측과 사전 연결 없이, 목적지 주소 등 전송에 필요한 모든 정보를 스스로 담고 있는 독립적인 데이터 단위입니다.
    
- 비연결성
    - 데이터를 보내기 전에 handshake를 거치지 않고 일단 목적지를 향해 던지는 방식입니다.
    
- 비신뢰성
    - 데이터가 순서대로 도착하는지, 중간에 유실되었는지 확인하지 않지만 전송 속도가 매우 빠릅니다.

##### Best-effort delivery

- 책임 회피형 전달
    
    - 네트워크가 데이터를 목적지까지 전달하기 위해 최선을 다하지만, 도착 여부나 데이터의 상태를 보장하지 않는 방식입니다.
        
- 단순한 메커니즘
    
    - 복잡한 에러 복구나 재전송 로직이 없어 구조가 단순하며, 네트워크 장비(라우터 등) 부하를 최소화합니다.
        
- 상위 계층의 몫
    
    - 전송 중에 데이터가 사라지거나 순서가 뒤바뀌어도 네트워크(IP계층)는 상관하지 않으며, 이 문제를 해결하려면 상위 계층(TCP 등)에서 별도의 로직을 처리해야 합니다.
        

##### QUIC

- 개요
    
    - UDP를 기반으로 동작하면서, 그 위에 TCP의 신뢰성 전송 기능과 TLS의 보안 기능을 직접 구현한 계층 결합형 프로토콜입니다.
        
- 특징
    
    - HTTP/3의 근간이 되는 기술로 사용됩니다.
        

#### 2. 네트워크 제어 메커니즘

##### 흐름 제어 (Flow Control)

- 정의
    
    - 송신자가 수신자의 처리 속도를 넘어서지 않게 조절하는 것입니다.
        
- 목적
    
    - 송신 측이 데이터를 너무 빨리 보내서 수신 측의 Buffer가 넘치는(Overflow) 것을 방지합니다.
        
- 대상
    
    - 송신자(Sender)와 수신자(Receiver) 사이의 1:1 관계입니다.
        
- 주요 기법
    
    - Stop-and-Wait: 패킷 하나를 보내고 확인 응답(ACK)을 받을 때까지 기다린 후 다음 패킷을 보냅니다.
        
    - Sliding Window: 수신 측이 한 번에 받을 수 있는 데이터 양(Window Size)을 알려주면, 송신 측은 ACK 없이도 그만큼 데이터를 연속해서 보낼 수 있습니다.
        

##### 혼잡 제어 (Congestion Control)

- 정의
    
    - 네트워크 망의 데이터 처리 능력을 넘어서지 않게 조절하는 것입니다.
        
- 목적
    
    - 네트워크 내의 라우터나 스위치가 감당할 수 없을 정도로 많은 데이터가 몰려 패킷이 지연되거나 유실되는 것을 방지합니다.
        
- 대상
    
    - 송신자와 수신자뿐만 아니라, 그 사이를 잇는 네트워크 전체(Router 등)를 고려합니다.
        
- 주요 기법
    
    - Slow Start: 전송 속도를 아주 조금씩(지수적으로) 늘려가며 네트워크 상태를 확인합니다.
        
    - Congestion Avoidance: 혼잡이 예상되는 지점부터 속도를 선형적으로 완만하게 올립니다.
        
    - Fast Retransmit / Fast Recovery: 패킷 손실이 감지되면 즉시 전송 속도를 줄이고 빠르게 복구합니다.
        

##### 오류 제어 (Error Control)

- 정의
    
    - 데이터가 깨지거나 유실되었을 때 이를 탐지하고 복구하는 것입니다.
        
- 목적
    
    - 전송된 데이터가 전송 전과 동일하다는 것을 보장합니다.
        
- 주요 기법
    
    - Checksum: 데이터 전송 전후의 합계를 비교하여 데이터 손상 여부를 확인합니다.
        
    - ARQ(Automatic Repeat Request): 오류가 발생했거나 패킷이 유실되었을 때 재전송을 요청합니다.
        

#### 3. 개발 및 웹 개념

##### 파싱 (Parsing)

- 정의
    
    - 일련의 문자열을 분석하여, 컴퓨터가 이해하고 처리하기 쉬운 구조적 데이터로 변환하는 과정을 의미합니다.
        
- 파서 (Parser)
    
    - 이 작업을 수행하는 프로그램이며, 컴파일러나 인터프리터의 핵심 구성 요소 중 하나입니다.
        

##### 멱등성 (Idempotency)

- 정의
    
    - 연산을 여러 번 적용하더라도 결과가 달라지지 않는 성질을 의미합니다.
        

##### 보안 및 통신

- CORS (Cross-Origin Resource Sharing)
    
    - 추가적인 HTTP 헤더를 사용하여, 한 출처에서 실행 중인 웹 애플리케이션이 다른 출처의 선택한 자원에 접근할 수 있는 권한을 부여하도록 브라우저에 알려주는 체제입니다.
        
- 프리플라이트 (Preflight)
    
    - 실제 요청을 보내기 전, 브라우저가 예비 요청(OPTIONS 메서드)을 보내서 서버가 해당 요청을 허용하는지 확인하는 과정입니다.
        

#### 4. 서비스 및 소프트웨어 구조

##### SaaS (Software as a Service)

- 정의
    
    - 클라우드 제공업체가 애플리케이션 소프트웨어를 호스팅하고, 사용자가 인터넷을 통해 이를 사용할 수 있도록 하는 클라우드 컴퓨팅 형태입니다.
        
- 특징
    
    - 웹 기반 접근성: 별도의 설치 과정 없이 웹 브라우저만 있으면 언제든 접속 가능합니다.
        
    - 구독 모델: 한 번에 큰 비용을 내는 것이 아니라 매달 또는 매년 사용료를 지불하는 방식입니다.
        
    - 자동 업데이트: 공급업체가 서버에서 업데이트를 진행하여 모든 사용자가 즉시 최신 버전을 사용합니다.
        
    - 멀티 테넌시: 하나의 소프트웨어를 수많은 사용자가 공유하되 각 데이터는 철저히 분리되어 보호받습니다.
        

##### 애플리케이션 (Application)

- 정의
    
    - 운영체제 위에서 실행되며 사용자가 특정 업무나 목적을 수행할 수 있도록 돕는 소프트웨어입니다.
        
- 실행 환경 분류
    
    - 데스크톱 앱: PC나 노트북 OS에 설치하여 사용 (MS Office, 포토샵 등)
        
    - 모바일 앱: 스마트폰, 태블릿 OS 전용으로 제작 (인스타그램, 유튜브 등)
        
    - 웹 앱: 설치 없이 웹 브라우저를 통해 실행 (Google Docs, Notion 등)
        
- 현대적 구조
    
    - 프레젠테이션 계층, 비즈니스 로직 계층, 데이터 계층으로 구분됩니다.
        

##### 컴퓨팅 (Computing)

- 정의
    
    - 컴퓨터 기술을 사용하여 데이터 처리, 문제 해결, 정보 생성 등 모든 활동을 의미합니다.
        
- 3대 핵심 요소
    
    - 입력 (Input), 처리 (Processing), 출력 (Output)
        
- 기술 트렌드 분류
    
    - 클라우드, 에지, 분산, 양자 컴퓨팅 등
        

#### 5. 최신 트렌드

##### 바이브 코딩 (Vibe Coding)

- 정의
    
    - 엄밀한 설계나 논리적 분석보다는 느낌(Vibe)과 직관에 의존하여 코드를 작성하는 방식을 일컫는 신조어입니다.
        
- 특징
    
    - 최근 AI를 활용하여 구체적인 로직 구현을 AI에게 맡기고, 개발자가 전체적인 방향성만 설정하며 코딩하는 방식을 의미하기도 합니다.
