## 2. 발생 이슈 및 해결 과정 (Troubleshooting)

### **이슈 A: 한글 인코딩 깨짐 현상**

- **증상:** 결과 화면의 '띠', '디데이' 및 알림창의 한글이 `ï¿½`와 같은 외계어로 출력됨.
    
- **원인:** 파일 저장 시 **UTF-8**이 아닌 ANSI 등 다른 인코딩 형식이 적용되었거나, 난독화 과정에서 한글 문자열이 손상됨.
    
- **해결:** 난독화된 자바스크립트 배열 내의 모든 한글 문자열을 **유니코드 이스케이프 시퀀스(`\uXXXX`)**로 변환하여 파일 인코딩 환경과 무관하게 정상 출력되도록 조치함.
    

### **이슈 B: 잘못된 입력값에 의한 계산 오류 (D-NaN)**

- **증상:** `920002`(존재하지 않는 0월) 등 유효하지 않은 생년월일 입력 시 다음 생일까지 남은 날짜가 `D-NaN`으로 표시됨.
    
- **원인:** 자바스크립트 `Date` 객체가 잘못된 날짜를 처리하는 과정에서 숫자가 아닌 값(NaN)을 반환함에도 불구하고 별도의 예외 처리가 부재함.
    
- **해결:** 입력받은 연/월/일을 기반으로 생성된 `Date` 객체가 실제 달력상에 존재하는 날짜인지 검증하는 **데이터 유효성 검사(Validation)** 로직을 추가하여 잘못된 입력 시 경고창을 띄우도록 수정함.
    

### **이슈 C: 난독화 후 실행 불가 (Runtime Error)**

- **증상:** 코드 보호를 위해 난독화를 적용한 후 '계산하기' 버튼을 눌러도 반응이 없음.
    
- **원인:** 난독화 과정에서 HTML 요소의 ID값(`birthInput`)에 오타가 발생하거나, 함수명이 일치하지 않아 참조 오류(Reference Error)가 발생함.
    
- **해결:** 코드 감사를 통해 오타를 수정하고, HTML의 `onclick` 이벤트와 자바스크립트의 난독화된 함수명을 재동기화하여 정상 작동 확인.
