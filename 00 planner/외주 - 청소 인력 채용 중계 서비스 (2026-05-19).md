

### 미팅 2026-05-19

- **참고**
    - 기존 사업체와는 별개의 시범 사업.
    
-  **서비스 목적**
    - 외국인 또는 한국인 청소 인력 채용 중계 앱 또는 웹
        - 초기 콘셉트 : 개인용 구인 공고 서비스
        - 스케일업 이후 콘셉트 : 호텔 청소, 룸메이드, 에어앤비 청소 세가지가 주요 채용 항목 B2B, B2C
      
- **타게팅**
    - 1순위 : 외국인, 몽골인
    - 2순위 : 한국인
    
- **이슈 사항**
    - 서비스를 구현하더라도 홍보가 되어야 사람들이 이용할 수 있기 때문에 홍보에 대한 고민.
    - 마케팅을 어떻게 할 것인가?
        - 주변 지인들을 통해 입소문 홍보.
        - 페이스북 : 외국인들은 대부분 페이스북을 이용하며 이 곳에서 홍보 및 공고를 구하는 전략을 이미 사용 중.
        - 홍보는 인하형에게 맡기면 된다.
    
- **기능 요구 사항**
    - 웹 또는 앱 둘다 가능하다.
        - 가능하면 웹으로 가는게 좋다고 이야기가 나옴.
        - 앱을 할 경우 안드로이드, IOS 둘다 지원
        - 웹을 할 경우 반응형도 구현.
    - 회원가입, 로그인, 회원 관리
        - 이름
        - 연락처
        - 비자(이건 회원가입에 입력해도 되는 데이터인지 알아봐야함)
        - 나이
        - 성별
        - 이미지 사진 첨부(지원자 얼굴, 업체 이미지, 장소 등등)
    - 게시판
        - 초기
            - 카테고리 : 공고 모집 게시판
                - 운영자가 공고를 올리면 사용자들이 그 공고를 확인 후 지원하는 방식.
            - 카테고리 : 고객문의 게시판
        - 스케일업
            - 청소 구인 게시판
            - 인력 구직 게시판
            - 지원자 리스트
            - 관리자 페이지 (검토)
    - 언어
        - 한국어, 영어 지원
    
- **비용 및 인수 인계**
    - 구현 및 유지 보수 일정 기간 2개월간 무료
    - 추후 유지 보수 및 관리는 별도 비용 또는 외주를 통해 받을 것을 통지함.
    - 형님은 나에게 지분을 갖되, 사이트를 내가 전적으로 관리하는 것도 권함.
    
- **개발 기간** 
    - 1달 ~ 2달? 
    - 더 걸릴 수도 있음
    - 무료로 해주는 것이니 압박이나 부담은 없지만 너무 늘어지지 않도록 주의해야한다.
    
- **ETC**
    - 글작성할때는 그 지원자의 이력서처럼 필수정보가 조회되도록.
    - 초기 콘셉트는 인력 모집으로써 운영자가 인력을 꽂아주는 형식으로 가되, 볼륨이 커지면 b2b, b2c 방향으로 전환.
    - 서버, DB : 무료로 최대한 방향성 가지는데, 추후 유료 서버 또는 DB로 가능성도 염두
    - 보안 : 
    - 가이드라인 인수인계 문서 작성

    - **사이트를 아예 내가 양도를 받고 지분을 받을 것인지? 아예 구현 및 일정 기간 유지 보수만 하고 손을 뗄 것인지 견해를 듣고 싶습니다.**
        - 잠재적으로 잘되면 수익을 기대할 수 있지만 당연히 안될 수도 있다, 유지보수 및 관리는 내가 다 책임져야합니다.
        - 다만, 인하형은 진지하면서도 실험적인 차원에서 사업으로 생각하고 있어서 가볍게 생각하진 않을 것 같습니다.
        
    - **서비스를 웹으로 만들것인지, 앱으로 만드는 도전을 하는게 좋을 것인지 견해를 듣고 싶습니다.**
        - 앱이 아닌 웹으로 최소 기능만 구현하는 서비스를 만들기로 정해졌습니다. 
        - 운영자가 구인 공고 올리면 사용자들이 지원하는 형태로 초기콘셉트를 가지되, 나중에 스케일업이 되면, b2b 방식으로도 고려하기로 했습니다.
        - 웹으로 개발하면 익숙하고 더 빨리 끝낼 수 있겠지만, 시간을 투자하기에는 앱에 비해 가치가 적을 것 같습니다.
        - 앱으로 개발을 한다면 이번 기회에 앱이란 것도 처음 개발해보며 익숙하지 않았던 다양한 기술 스택을 경험하는 것도 좋을 것 같습니다만, 러닝커브 굴곡이 클 것으로 예상되며, 조금 모험하는 기분이라 걱정이 드는 것도 사실입니다.

### MVP 출시 타임라인 2026-05-20

#### 1주차 : 설계 및 인프라 구축 (뼈대 잡기)
- **목표** : 데이터베이스 구조를 확정하고, 무료 서버 환경을 세팅한다.
      
- **주요 태스크**
    - **DB 모델링** : 회원(User), 공고(Job), 지원서(Application) 테이블 설계
    - **인프라 구축** : AWS 프리티어 또는 Supabase/Vercel을 활용한 개발 환경 세팅
    - **다국어(i18n) 기획** : 한국어/영어 번역 텍스트를 관리할 구조 설계
    
- **산출물** : DB ERD(다이어그램), API 명세서 초안
      
- **비고**
    - 비자(Visa) 정보는 회원가입 테이블에서 제외하고, 우선 `지원서 작성 단계`에서 텍스트로 입력받도록 DB를 설계한다. 그래야 초기 가입 허들이 낮아진다.
---
#### 2주차 : 백엔드 핵심 기능 개발 (기능 구현)
- **목표** :  회원인증 및 운영자용 공고 관리 API를 완성한다.
      
- **주요 태스크**
    - **인증(Auth) 시스템:** 이메일/비밀번호 또는 카카오/페이스북 간편 로그인 구현
    - **공고(Job) API:** 운영자가 공고를 등록, 수정, 삭제, 조회(CRUD)할 수 있는 기능 개발
    - **이미지 업로드 기능:** AWS S3 프리티어 등을 연동하여 업체 이미지 업로드 API 구현
    
- **산출물** : 백엔드 API 서버 배포 완료
---
#### 3주차 : 프론트엔드 UI/UX 및 반응형 레이아웃 (시각화)
- **목표** : 모바일(페이스북 인앱 브라우저) 화면에 최적화된 반응형 웹 스켈레톤을 만든다.
    
- **주요 태스크:**
    - **공통 컴포넌트:** 네비게이션 바, 푸터, 다국어 전환 버튼 구현
    - **반응형 레이아웃:** 테일윈드(Tailwind CSS) 등을 활용해 모바일 화면 위주로 퍼블리싱
    - **페이지 구현:** 메인 페이지(공고 리스트), 로그인/회원가입 페이지
    
- **산출물** : 프론트엔드 UI 화면 개발 완료
---
#### 4주차 : 데이터 연동 및 기능 결합 (풀스택 완성)
- **목표** : 백엔드 API와 프론트엔드 UI를 연결하여 실제 작동하는 서비스를 만든다.
    
- **주요 태스크**
    - **API 연동** :  프론트엔드에서 공고 리스트 호출 및 회원가입 기능 연결
    - **지원서 제출 시스템** :  사용자가 공고를 보고 '지원하기' 버튼을 눌렀을 때, 필수 정보(이름, 연락처 등)가 운영자 DB로 쌓이도록 연동
    - **다국어 적용** :  버튼 클릭 시 한/영 텍스트가 실시간으로 변환되도록 프론트엔드 로직 완성
    
- **산출물** :  주요 기능이 정상 작동하는 웹 어플리케이션
---
#### 5주차 : 운영자(관리자) 페이지 및 디버깅 (안정화)
- **목표:** 지인(운영자)이 사용할 관리자 화면을 만들고, 전체적인 버그를 잡는다.
    
- **주요 태스크**
    - **어드민 대시보드** : 운영자가 로그인했을 때, 지원자 리스트를 한눈에 볼 수 있는 심플한 페이지 구현
    - **크로스 브라우징 테스트** : 특히 스마트폰 페이스북 앱 내부 브라우저에서 화면이 깨지지 않는지 집중 테스트
    - **디자인 디테일 수정** : 톤앤매너(심플한 블루 컬러) 맞추기
    
- **산출물:** 관리자 기능 포함 통합 테스트 완료 버전
---
#### 6주차 : 최종 배포 및 런칭 (인수인계)
- **목표** : 상용 서버에 도메인을 연결하고 서비스를 공식 오픈한다.
    
- **주요 태스크**
    - **최종 배포** : 실서버 배포 및 도메인 연결
    - **가이드라인 문서 작성** : 운영 방법(공고 올리는 법, 지원자 확인법)을 담은 매뉴얼 작성 및 지인에게 전달
    - **마케팅 시작 연동** : 페이스북 광고/홍보 링크가 정상적으로 작동하는지 최종 확인
    
- **산출물** : 라이브 서비스 URL, 인수인계 가이드 문서

---

#### 이후 스케일업 단계 (출시 후 2달간의 무료 유지보수 기간)
이 기간에는 새로운 기능을 무리하게 추가하기보다, **안정성 확보**와 **데이터 수집**에 집중해야 한다.

- **1단계 (지표 확인)** : 페이스북 홍보를 통해 실제 외국인 유저들이 유입되는지 확인 (GA 등 간단한 분석 툴 심어두기).
- **2단계 (지분 협상 시점)** : 유저가 모이고 비즈니스 가능성이 보인다면, 6주차 끝나는 시점에 정식 지분 계약서(혹은 수익 배분 조건) 작성을 요구.
- **3단계 (B2B 확장)** : 서비스 볼륨이 커지면 그때 앱(App) 전환이나 호텔/룸메이드 자동 매칭 시스템(B2B) 아키텍처 설계를 시작하면 된다.
