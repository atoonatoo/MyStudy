## 1. 프로젝트 기획: 왜 '마이크로 웹 도구'인가?

바이브 코딩의 핵심은 AI와의 대화로 복잡한 설계를 건너뛰고 **'동작하는 결과물'**을 즉시 뽑아내는 것입니다. 대규모 서비스는 AI만으로 한계가 있지만, 특정 기능을 수행하는 단일 페이지 웹앱은 바이브 코딩에 완벽히 부합합니다.

### **1.1 대상 아이템 (Micro Web Tools)**

- **유틸리티:** 단위 변환기, 이자 계산기, 로또 번호 생성기, 만 나이 계산기.
    
- **개발/업무:** JSON 포맷터, SQL 쿼리 정리기, 글자수 세기, 비밀번호 생성기.
    
- **엔터테인먼트:** 오늘 뉴스 요약(API 연동), MBTI별 대화법, 랜덤 점심 메뉴 추천.
    

### **1.2 5대 원칙 준수 여부**

- **비용:** Vercel/GitHub(무료), 제미나이(기존 구독 활용) → **완전 무료**
    
- **법적 안전:** 사용자 데이터를 서버에 남기지 않는 **Client-side** 로직 → **안전**
    
- **수익성:** 방문자 수 기반 **광고/제휴 수익** → **수익 발생**
    
- **생산성:** HTML 하나로 끝나는 **Single-File** 방식 → **대량 생산**
    
- **유지보수:** 서버 없는 **Serverless** 구조 → **유지보수 용이**

## 2. 기술적 구현: 제미나이 최적화 바이브 코딩

커서(Cursor) 같은 유료 도구 없이도 제미나이의 강력한 성능을 활용하여 생산성을 극대화하는 방식입니다.

### **2.1 Single-File Artifact 전략**

복잡한 프로젝트 구조(React, Vue 등)를 지양하고, 하나의 `.html` 파일에 모든 로직(HTML, CSS, JS)을 담는 방식입니다.

- **생산성:** 제미나이가 출력한 코드 블록을 한 번만 복사해서 붙여넣으면 끝납니다.
    
- **이동성:** 파일 하나만 관리하면 되므로 파일 참조 오류가 발생하지 않습니다.
    
- **배포:** Vercel 등에 업로드할 때 추가 빌드 설정이 필요 없습니다.
    

### **2.2 제미나이 컨텍스트 윈도우(Context Window) 활용**

제미나이는 매우 넓은 '작업 메모리'를 가지고 있습니다. 이를 활용해 '공장장' 역할을 수행하게 합니다.

- **학습:** 이전에 만든 성공적인 도구의 코드와 디자인 가이드를 제미나이에게 한 번에 입력합니다.
    
- **변주:** "앞서 만든 계산기 템플릿의 디자인을 유지하면서, 이번엔 '부동산 취득세 계산기' 로직으로 바꿔줘"라고 요청합니다.
    
- **일관성:** 수십 개의 프로젝트를 진행해도 동일한 퀄리티와 브랜드 아이덴티티를 유지할 수 있습니다.

## 3. 운영 전략: 수익 파이프라인(Pipeline) 구축

개별 사이트를 만드는 데 시간을 쏟는 대신, **아이디어가 웹사이트로 변하는 공정**을 자동화해야 합니다.

### **3.1 배포 자동화 (GitHub + Vercel)**

1. **Repository 구성:** 하나의 깃허브 저장소 안에 각 도구를 폴더별로 관리하거나, 프로젝트별로 별도 생성합니다.
    
2. **자동 배포:** 코드를 푸시(Push)하는 즉시 전 세계에 배포되는 파이프라인을 구축합니다.
    
3. **도메인 관리:** 하나의 메인 도메인을 구매한 후, 서브도메인(`tool1.mysite.com`, `tool2.mysite.com`)을 무한 생성하여 비용을 절감합니다.
    

### **3.2 수익 채널의 단계적 확장**

- **1단계 (즉시 수익):** 제휴 마케팅 (쿠팡 파트너스, 아마존 어소시에이트). 도구 하단에 관련 상품 링크를 배치합니다. 승인 절차가 없어 즉시 시작 가능합니다.
    
- **2단계 (안정적 수익):** 구글 애드센스. 도구가 10~20개 정도 모이고 SEO 점수가 쌓이면 신청합니다.
    
- **3단계 (네트워크 광고):** 애드센스 승인이 어려울 경우 Adsterra 등을 활용하되, 반드시 **광고 필터링 설정**을 통해 저품질 광고를 차단하여 사이트 평판을 보호합니다.

## 4. 리스크 및 트레이드 오프 (Trade-off) 분석

효율적인 수익 활동을 위해 우리가 수용해야 할 명확한 손익 계산입니다.

|**항목**|**얻는 것 (Gain)**|**포기하는 것 (Pain/Trade-off)**|
|---|---|---|
|**개발 방식**|압도적인 속도와 무지성 생산성.|정교한 커스텀 기능과 독보적인 디자인.|
|**호스팅**|서버 비용 0원, 무한 확장성.|서버 측 로직(DB 저장, 회원가입 등) 사용 불가.|
|**광고 선택**|승인 없는 즉각적인 수익화.|저품질 광고 노출 시 SEO(검색 순위) 하락 위험.|
|**기술 스택**|제미나이 하나로 모든 공정 해결.|커서(Cursor)의 자동 코드 적용 편의성.|

## 5. 최종 로드맵 및 행동 지침

### **STEP 1: 만능 템플릿 제작 (Day 1)**

제미나이에게 **"SEO에 최적화되고, 광고 공간이 확보되었으며, 다크 모드가 지원되는 단일 페이지 웹 도구용 HTML 템플릿"**을 짜달라고 하세요. 이것이 당신 공장의 기본 프레임이 됩니다.

### **STEP 2: 아이템 선정 및 양산 (Day 2~7)**

키워드 분석 도구를 통해 사람들이 많이 찾지만 웹사이트가 허술한 키워드를 5개 선정합니다. 제미나이를 통해 하루에 하나씩 도구를 출시합니다.

### **STEP 3: SEO 및 마케팅 (Ongoing)**

도구 하단에 AI가 작성한 **"이 계산기가 필요한 이유", "사용 방법"** 등 풍부한 텍스트 콘텐츠를 채워 넣으세요. 검색 엔진은 텍스트가 많은 페이지를 사랑합니다.

### **STEP 4: 수익 최적화 (After 1 Month)**

방문자가 발생하는 페이지를 분석하여 광고 위치를 조정하고, 성과가 좋은 도구는 더 고도화된 기능을 추가합니다.
