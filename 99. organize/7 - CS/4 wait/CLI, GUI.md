##### 제목 : 

- Terminal
    
    ```java
    - 터미널이란? : 
    	유닉스 기반의 대형 컴퓨터에서 사용자들이 컴퓨터에 접속하기 위해 사용했던 단말기로 개인용 컴퓨터가 보급된
    	요즘에는 "컴퓨터를 직접 제어하기 위해 사용하는 소프트웨어"를 뜻한다.
    	
    	우리가 익숙한 그래픽으로 이루어진 GUI와 텍스트 기반인 CLI(명령어로 컴퓨터를 조작하는 방식)를 통해 
    	터미널을 사용하는 이유는 "일부 작업을 완료하는 것이 훨씬 빠르고, 
    	더 많은 명령과 스크립트에 접근할 수 있	때문이다. 
    	사용자가 컴퓨터를 조작하기 위해 명령어를 터미널에 입력하면, 쉘(보통은 Bash)이 해당 명령어를 해석하고 
    	커널이 이를 실행하는 구조이다.
    
    	위에서 언급한 것처럼 터미널은 CLI 방식으로 사용하고 하는 기능의 명령어를 입력해야 한다.
    	그러나 윈도우와 유닉스 명령어는 서로 다르기 때문에 각 OS의 맞는 명령어를 사용해야 한다.
    
    - 쉘(Shell) : 
    	쉘은 조개 껍데기의 의미로 '터미널을 사용하기 위한 소프트웨어 환경'을 의미한다.
    	키보드 입력과 명력을 실행할 수 있고 디렉토리와 파일 관리 기능을 지원한다.
    	쉘은 "커널과 사용자의 사이"에 존재한다.
    	사용자가 커널을 사용하기 위해 명령어를 입력하면 커널이 읽을 수 있는 이진코드로 해석해 준다.
    
    - 쉘의 종류 : 
    	* Windows shell : 
    		1) cmd : cmd는 Command의 약자로 절차지향적 성격을 가지고 있으며 윈도우NT계열에서 만들었다.
    		2) Power shell : Power shell은 .NET FreamWork라는 객체를 사용하며(객체지향적) 
    			 모든 결과들이 객체로 표현된다.
    
    	* Unix shell : 
    		1) sh : 초기의 유닉스 쉘(Bourne Shell), 1977년에 발표
    		2) ksh : 콘 쉘이라고도 불리며 sh의 확장판
    		3) csh : c언어를 기반으로 만든 쉘
    		4) bash : sh와 대부분 호환되는 쉘
    		& 리눅스와 MacOS는 유닉스 커널을 사용해서 유닉스 쉘과 일치한다.
    
    - 커널(kernel) :
    	모든 OS들은 커널을 가지고 있다.
    	커널은 OS계층이며 "HW와 컴퓨터에서 돌아가는 프로그램을 연결"시켜주는 역할을 한다. 
    	커널은 OS의 핵심이며 컴퓨터를 부팅시키면 가장 먼저 메모리에 올라오고 그리고는 컴퓨터가 꺼질때 까지 
    	내내 메모리에 상주하게 된다. 
    	커널은 모든 어플리케이션에게 인터페이스를 제공하며 HW를 컨트롤하고 프로세스들이 서로 정보를 주고
    	받는 것을 허락해준다.
    ```
    
    
- CLI / GUI
    
    ```java
    - 명령줄 인터페이스 (CLI) / 그래픽UI (GUI) : 
    	"명령줄 인터페이스는 키보드를 이용해 운영 체제와 상호 작용하는 소프트웨어 메커니즘이다."
    	또 다른 메커니즘인 그래픽 UI(GUI)는 오늘날 모든 어플리케이션과 소프트웨어 시스템에서 자주 사용된다. 
    	GUI를 사용하면 아이콘과 이미지를 시각적으로 탐색하고 클릭하여 작업을 수행할 수 있다. 
    	그러나 GUI는 시스템 관리 작업에 비효율적이며, 가상 또는 원격에서는 더욱 효율이 떨어진다. 
    	명령줄 인터페이스를 사용하면 텍스트 명령을 입력하여 모든 서버 또는 컴퓨터 시스템에서 프로그램을 구성, 탐색
    	또는 실행할 수 있다. 
    	Linux, macOS와 Windows를 포함한 모든 운영 체제는 시스템과 더 빠르게 상호작용할 수 있는 CLI를 제공한다.
