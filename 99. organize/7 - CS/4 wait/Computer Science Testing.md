##### 제목 : 

- 유닛 테스트
    
    ```java
    - 유닛테스트 | 유당테스트 : Unit Test
    
    - 목적 : 프로그램의 각 부분을 고립 시켜서 각각의 부분이 정확하게
             잘 동작하는지 확인하는 것
      [함수나 메소드 등을 분리하여 기능이 잘 작동하는지 고립시켜서 테스트하는 것]
    ```
    
- 통합 테스트
    
    ```java
    - 통합테스트 : Intergration Test
    
    - 어플리케이션의 여러 컴포넌트나 모듈이 결합되었을 때 올바르게 작동하는지
    	확인하는 테스트
    
    - 유닛 테스트에서 테스트된 유닛들이 결합되어있을 때 올바르게 작동하는지 확인
    
    - 즉 각 기능들이 모두 결합된 만들고자한 프로그램의 자체를 테스트하는 것이며
    	모든 기능들이 원할하게 작동하는지 확인하는 것
    
    - 단위 테스트와 시스템 테스트를 보완하여 소프트웨어의 전반적인 품질과 안정성을
    	보장하는데 도움이 되는 포괄적 테스트 전략이다.
    
    - 통합테스트의 주요 관점 : 
    	* 컴포넌트의 상호작용 : 예상대로 통신하고 데이터를 공유하는지 확인한다.
    	* API 및 서비스 유효성 검사 : 외부 API 혹은 타사 컴포넌트가 어플리케이션
    		컨텍스트 내부에서 올바르게 상호작용하는지 확인한다.
    	* 데이터 일관성 : 여러 컴포넌트, 외부 서비스가 시스템 전체에서 데이터를 
    		일관적으로 처리하는지 확인한다.
    	* 기능 흐름 : 전반적인 기능의 흐름이 End-To-End에서 의도한대로 
    		작동하는지 확인한다.
    
    - 통합테스트의 접근 방식 : 
    	* 빅뱅(Big Bang) : 
    		1. 모든 구성요소와 모듈을 한번에 통합하고 전체 시스템으로 테스트한다.
    		2. 시간이 많이 걸리고 어려울 수 있다는 단점이 있다.
    	* 점진적(Incremental) : 
    		1. 구성 요소를 하향식 혹은 상향식으로 혹은 이 두가지 방법을 합쳐(샌드위치)
    			 점진적으로 테스트한다.
    		2. 테스트 범위가 더 좁아져서 테스트하기가 수월해진다.
    ```
    
- 인수 테스트
    
    ```java
    - 인수 테스트 | 수락 테스트 : Acceptance Test
    
    - 소프트웨어 개발 프로세서에서 최종 사용자 혹은 이해관계자가 지정한 요구 사항을 
    	충족하였는지 확인하기 위해 시스템을 테스트하는 단계이다.
    
    - 주요 목적은 소프트웨어가 비지니스 요구 사항을 충족하고, 합의된 사항을 준수하머
    	배보할 준비가 되었는지 확인하는 것이다.
    
    - 보통 단위 테스트나 통합 테스트와 같은 다른 테스트 단계 이에 최종적으로
    	진행되는 테스트이.
    
    - 특정 결함이나 문제를 식별하기보다는 시스템의 기능, 사용성, 성능 및 사용자
    	기대치와의 전반적인 일치 여부를 평가하는데 중점을 둔다.
    
    - 인수 테스트가 이뤄지는 과정 : 
    	* 최종사용자 혹은 이해관계자가 시스템을 사용하여 자신의 요구사항과 기대치를
    		충족하는지 확인한다.
    	* QA 팀은 사전 정의된 테스트 케이스 혹은 시나리오를 진행한다.
    	* 전문적 QA팀과 최종 사용자는 각각 사용자 관점 및 기술 전문성에 대한 테스트를
    	  조합하여 균형을 유지한다.
    ```
