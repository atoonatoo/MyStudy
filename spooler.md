- 정의
	- Spooler는 Harvester가 수집한 로그 이벤트를 일정량 모아둔 뒤, 배치(batch) 단위로 libbeat로 넘겨주는 임시 저장 버퍼이다.

- 특징
	- 이벤트 버퍼
		- 로그를 한줄씩 바로 보내지 않고 모아두었다가 일정 조건이 되면 전송
	- 성능 최적화
		- 이벤트마다 전송하면 느려지기 때문에 여러 줄 모아서 한 번에 처리한다.(Batching)
	- 메모리 내 처리
		- 디스크가 아닌 메모리 기반, 고속
	- 전송 전 처리 단계
		- 이 단계에서 Processor 적용도 가능하다.
