### 현실적 주간 플랜 예시

| 요일    | 시간                           | 내용                       |
| ----- | ---------------------------- | ------------------------ |
| 월·화·수 | 퇴근 후 1시간                     | 업무 복습, 에러 로그 정리, 개념 리마인드 |
| 목     | 휴식 또는 가벼운 블로그 정리             |                          |
| 금     | 일주일 업무 회고, 개선점 기록            |                          |
| 토     | 개인 공부 2~3시간 (공식 문서/토이 프로젝트)  |                          |
| 일     | 완전 휴식 또는 코드 리뷰 읽기 (Github 등) |                          |

- 핵심은 `루틴화`지 `몰입학습`이 아니다.
- 한 번에 몰아서 하면 2주 후엔 번아웃이 온다.
- **하루 1시간을 1년 지속하는 게 10시간 몰입보다 훨씬 강하다.**

### 5분 루틴 예시 세트 (하루 구성 예)
|시간대|예시 루틴|습관 효과|
|---|---|---|
|오전 출근 전|기술 단어 1개 암기|머리 워밍업|
|점심 직후|GitHub Trending README 읽기|기술 감각 유지|
|오후 커피 타임|회사 코드 한 함수 읽기|실무 감각 강화|
|퇴근 전|1문장 회고 작성|기록 루틴 확립|

---
### 1단계 : 회사 일에 흡수되되, 구조를 이해하는 3개월

- **목표** 
    - "내 코드가 어디에, 왜 쓰이는지”를 정확히 파악하기  
- **키워드** 
    - 구조 파악, 실무 언어 적응, 문제 기록

|구분|내용|
|---|---|
|**회사 중심 학습**|업무 중 만난 API, 서버 흐름, DB 구조를 이해 중심으로 메모하기 (암기 X, “흐름도”로 기록)|
|**업무 일지 작성**|매일 5줄이라도 “오늘 배운 기술, 막혔던 점, 해결 방법” 기록 — Obsidian이나 Notion 추천|
|**공부 방향**|회사에서 쓰는 기술 스택 위주로만 공부 (Spring Boot면 JPA, MySQL, Redis 등 해당 생태계 한정)|
|**시간 배분**|주중 1시간, 주말 3시간 이하 — 복습 중심으로 충분|

- **핵심 팁**
    - 아는 걸 새로 배우지 말고, **일하면서 만난 모르는 걸 메워라.**  
    - 이 시기엔 “업무 + 복습”이 전부이다.
---
### 2단계 : 기술 깊이 확장기 (4~8개월차)

- **목표** 
    - “회사 코드 바깥의 원리를 이해하기”  
- **키워드** 
    - 공식 문서, 미니 프로젝트, 로그·성능 감각

| 구분        | 내용                                                                                                                     |
| --------- | ---------------------------------------------------------------------------------------------------------------------- |
| **학습 비중** | 주중 1~1.5시간, 주말 하루 2~3시간 — _꾸준함이 핵심_                                                                                    |
| **공부 영역** | 1. 언어 심화(Java, JS 등 내부 동작 원리)<br>2. 백엔드라면 DB 트랜잭션, HTTP, 캐싱, 동시성 같은 기본기 <br>3. DevOps 기초 (Git, Docker, CI/CD, 로그 분석 등) |
| **실행법**   | 회사 코드의 일부 기능을 클론해서 로컬에서 다시 만들어보기 → 실무와 연결된 공부는 피로감이 훨씬 덜함                                                              |
| **출력물**   | 기술 블로그나 깃허브 정리 (외부 평가용 + 복습용)                                                                                          |
- **핵심 팁**  
    - 이 시기엔 “왜 이렇게 구현했을까?”를 매일 한 번씩만 물어보자
    - 그게 성장의 속도를 결정한다.

---

### 3단계 : 자기 브랜딩 및 성장 고도화 (9~12개월차 이후)

- **목표** 
    - “내가 다루는 기술로 문제를 해결할 수 있는 사람이라는 인식 만들기” 
      
- **키워드** 
    - 포트폴리오화, 발표/기록, 방향성 확립

|구분|내용|
|---|---|
|**포트폴리오 프로젝트**|회사 도메인과 무관하게 개인적으로 작은 SaaS, 블로그, 유틸리티 툴 하나 제작 (배포까지 경험)|
|**학습 루틴**|주 2회 1시간씩 → “공식 문서 읽기 + 코드 분석” 반복 (이 시기엔 깊이보다 “안정적인 리듬”)|
|**지식 정리**|Velog·GitHub README로 문서화 (리크루터가 실제 보는 포인트)|
|**브랜딩 요소**|“1년 차지만 꾸준히 성장 중인 사람”이라는 기록이 남아 있어야 다음 이직 때 강력한 무기|
