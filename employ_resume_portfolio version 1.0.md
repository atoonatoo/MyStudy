- Jmeter 선정 이유
    - Jmeter, k6, Locust, nGrinder들을 공식 문서를 통해 공부했다.
      
    - 공부하면 공부할 수록 Jmeter를 선택한 납득할 수 있는 이유를 찾기 어렵고, 다른 도구들은 선택에 대한 납득할 수 있는 이유가 풍부하게 느껴진다.
      
    - Jmeter : `입문자 친화적 테스트 도구`, `다양한 프로토콜 지원`,`풍부한 플러그인`, `GUI 특화된 테스트 도구`, `리포트 자동 생성`, `Java 호환성`, `GUI (XML)or CLI 기반 실행기`
    
    - k6 : `개발자 친화적 테스트 도구`, `가볍고 높은 성능 보장`, `통합성과 확장성`, `대규모 부하테스트에 유리`, `모니터링`, `코드 기반 실행기`, `Runner 확장`
    
    - nGrinder :  `조직 환경 친화적 테스트 플랫폼` , `결과 수집 및 리포팅`, `모니터링`, `메트릭 수집 및 시각화`, `지원언어 : Python or Groovy`, `중앙 괸리형 웹 콘솔 기반 실행기`, `내부 SYN 저장소로 스크립트 버전 관리`, `Controller-Agent 구조`
    
    - Locust : `코드 기반과 관리 UI의 하이브리드형`, `테스트 흐름의 유연성`, `조건 분기 자유도`, `Web UI 실시간 제어 및 모니터링`, `CLI 기반 실행기`, `Master-Worker 구조`
