## 1. 부하 테스트 설계와 도구 선정

### 1.1 k6
- k6는 Go 언어로 작성된 오픈소스 부하 테스트 도구이다. 클라우드 기반의 성능 테스트 플랫폼을 제공하는 Grafana Labs에서 개발했으며, 개발자 중심의 사용성을 제공하는 것이 특징이다.
- 장점
    - 스크립트 작성의 용이성
        - 테스트 시나리오 작성 시 JS(ES6) 문법을 사용
        - 대다수 개발자에게 친숙
        - 복잡한 로직이나 데이터 처리도 쉽게 구현 가능
          
    - 높은 성능과 효율성
        - 코어 부분이 Go 언어로 작성
        - 스크립트 언어를 사용함에도 불구하고 타 도구에 비해 메모리 효율성이 높고, CPU 사용류도 낮음
        - 많은 가상 사용자를 처리할 수 있음
          
    - 개발자 친화적인 워크플로우
        - 테스트 코드를 관리하고 버전 관리(Git)가 용이한 테스트 코딩 접근 방식을 지원
        - CLI 기반으로 설계되어 CI/CD 파이프라인에 쉽게 통합 가능
          
    - 다양한 결과 출력 및 통합
        - 다양한 형식의 테스트 결과를 출력 가능(JSON, CSV, InfluxDB ..)
        - Grafana 등의 모니터링 도구와 연동이 용이
        - 테스트 스크립트 내에서 응답 시간이나 성공률 등에 대한 임계값(Thresholds)을 설정하고, 이 임계값을 넘을 경우 테스트를 실패 처리할 수 있어 자동화된 검증에 효과적이다.
          
    - 확장성(확장모듈)
        - xk6라는 확장 프레임워크를 통해 HTTP뿐만 아니라, gRPC, Kafka, Browsertesting 등 다양한 프로토콜 기능에 대한 커스텀 모듈을 추가하여 테스트 가능
- 단점
    - 테스트 시라니오 복잡성
        - 스크립트 작성이 쉽지만, 테스트 로직이 매우 복잡하거나, 비기술적인 이해관계자가 테스트를 이해하기 위해 js 코드에 대한 이해가 필수적
    - GUI 도구 부재
        - JMeter와 같은 다른 도구들과 달리 GUI 제공하지 않으며, 모든 테스트 시나리오는 코드로 작성해야함
    - 브라우저 렌더링 테스트 제한
        - k6의 기본 기능은 헤드리스 환경의 프로토콜 부하 테스트에 중점을 둔다. 실제 웹 브라우저처럼 JS 실행, 이미지 로딩, CSS 렌더링 등이 모두 포함된 종단 간 테스트를 수행하려면 xk6-browser 확장 모듈을 별도로 사용해야 한다.
    - 부하 분산의 필요성
        - 단일 머신에서 처리할 수 있는 부하의 양에는 한계가 있어 대규모 부하 테스트를 위해 여러 머신에 걸쳐 테스트를 분산하는 분산 테스트 환경 구축이 필요
